# Tiny Second-hand Shopping Platform

화이트햇스쿨 Secure Coding 과제로 개발한 소규모 중고거래 플랫폼입니다.  
Flask와 SQLite를 사용했으며, 기본적인 중고거래 기능과 보안 기능을 함께 구현했습니다.

## GitHub Repository

https://github.com/lob1030/secure-coding

## 주요 기능

### 회원 기능

- 회원가입
- 로그인 및 로그아웃
- 프로필 소개글 수정
- 비밀번호 변경
- 비밀번호 해시 저장
- 비활성화된 사용자 로그인 차단

### 상품 기능

- 상품 등록
- 전체 상품 조회
- 상품 상세 조회
- 상품명과 설명 검색
- 본인 상품 수정 및 삭제
- 상품 이미지 URL 등록

### 사용자 소통 기능

- Flask-SocketIO 기반 전체 실시간 채팅
- 사용자 간 1대1 메시지

### 포인트 송금

- 회원가입 시 기본 포인트 지급
- 사용자 간 가상 포인트 송금
- 송금 내역 확인
- 잔액 부족, 음수 송금, 자기 송금 차단

### 신고 및 관리자 기능

- 사용자 신고
- 상품 신고
- 중복 신고 차단
- 신고 누적 시 사용자 또는 상품 자동 차단
- 관리자 페이지에서 사용자 차단 및 해제
- 관리자 페이지에서 상품 차단 및 해제
- 신고 내역 확인 및 삭제

## 설치 방법

### 1. 저장소 복제

```bash
git clone https://github.com/lob1030/secure-coding.git
cd secure-coding
```

### 2. Conda 환경 생성

```bash
conda env create -f enviroments.yaml
```

### 3. Conda 환경 활성화

```bash
conda activate secure_coding
```

이미 환경이 만들어져 있다면 환경 생성 과정은 다시 실행하지 않아도 됩니다.

## 실행 방법

보안을 위해 실행 전에 `SECRET_KEY` 환경변수를 설정하는 것을 권장합니다.

```bash
export SECRET_KEY=$(python -c "import secrets; print(secrets.token_hex(32))")
```

서버를 실행합니다.

```bash
python app.py
```

브라우저에서 다음 주소로 접속합니다.

```text
http://127.0.0.1:5000
```

처음 실행하면 현재 폴더에 `market.db` 파일과 데이터베이스 테이블이 자동으로 생성됩니다.

## 관리자 계정 설정

먼저 웹사이트에서 일반 계정을 회원가입합니다.

서버를 `Ctrl + C`로 종료한 뒤 다음 명령을 실행합니다.

```bash
python app.py --make-admin 사용자아이디
```

예시:

```bash
python app.py --make-admin test1
```

관리자 권한을 부여한 뒤 서버를 다시 실행합니다.

```bash
python app.py
```

관리자 계정으로 로그인하면 상단 메뉴에서 관리자 페이지를 이용할 수 있습니다.

## 보안 개선 내용

### 비밀번호 보호

기존 코드에서는 사용자의 비밀번호가 데이터베이스에 평문으로 저장되었습니다. 이를 수정하여 `generate_password_hash()`로 해시한 값을 저장하고, 로그인할 때는 `check_password_hash()`로 비교하도록 변경했습니다.

### SQL Injection 방어

사용자 입력값을 SQL 문자열에 직접 연결하지 않고 `?` 자리표시자를 사용하는 매개변수화 쿼리를 적용했습니다.

### CSRF 방어

상품 등록, 송금, 신고, 상품 삭제 등 상태를 변경하는 POST 요청에 CSRF 토큰을 추가했습니다. 서버에서 세션의 토큰과 폼으로 전달된 토큰을 비교합니다.

### 입력값 검증

다음 항목을 서버에서 검증합니다.

- 사용자 아이디 형식과 길이
- 비밀번호 길이와 영문·숫자 포함 여부
- 상품명과 설명 길이
- 상품 가격 범위
- 이미지 URL 형식
- 메시지 길이
- 송금 금액과 잔액

### 권한 검사

- 로그인하지 않은 사용자의 주요 기능 접근 차단
- 일반 사용자의 관리자 페이지 접근 차단
- 상품 소유자 또는 관리자만 상품 수정·삭제 가능
- 관리자 계정의 임의 차단 방지

### 채팅 사용자 사칭 방지

클라이언트에서 전달한 사용자명을 그대로 사용하지 않고, 서버 세션의 사용자 ID를 기준으로 실제 사용자명을 조회합니다.

### 송금 안전성

송금 시 다음 조건을 확인합니다.

- 송금액이 0보다 큰지 확인
- 받는 사용자가 존재하는지 확인
- 자기 자신에게 송금하는지 확인
- 보유 잔액보다 큰 금액인지 확인
- 오류 발생 시 `rollback()`으로 변경 취소

### 운영 정보 노출 방지

최종 실행에서는 Flask 디버그 모드를 비활성화했습니다.

```python
debug=False
```

## 개발 과정에서 발생한 문제

개발 환경을 구성하는 과정에서 Windows PowerShell과 Ubuntu 터미널을 혼동하여 Conda 명령이 실행되지 않는 문제가 있었습니다. 이후 Ubuntu 터미널에서 프로젝트 폴더로 이동한 뒤 `code .`으로 VS Code를 실행하여 해결했습니다.

상품 검색 기능을 처음 추가했을 때 검색창은 나타났지만 상품 목록이 필터링되지 않았습니다. HTML 화면만 수정하고 Flask의 `dashboard()` 함수에서 검색어를 처리하지 않은 것이 원인이었습니다. 서버 코드에서 `request.args`로 검색어를 받고 SQL `LIKE` 조건을 적용하여 수정했습니다.

서버를 여러 번 실행하면서 5000번 포트가 이미 사용 중이라는 오류도 발생했습니다. `lsof`로 포트를 사용하는 Python 프로세스의 PID를 확인하고 기존 프로세스를 종료한 뒤 다시 실행했습니다.

이 과정을 통해 화면 코드, 서버 코드, 데이터베이스가 서로 연결되어야 하나의 기능이 완성된다는 점을 확인했습니다.

## 한계 및 유지보수 방향

현재 프로그램은 과제와 로컬 테스트를 위한 소규모 플랫폼입니다.

- SQLite를 사용하므로 다수 사용자가 동시에 접속하는 운영 환경에는 한계가 있습니다.
- 상품 이미지는 실제 파일 업로드가 아니라 이미지 URL 방식입니다.
- 운영 환경에서는 HTTPS, 별도 데이터베이스 서버, 로그 관리가 추가로 필요합니다.
- 기능이 많아지면 `app.py`를 기능별 Blueprint로 분리할 필요가 있습니다.
- 자동화 테스트를 추가하면 코드 변경 후 기능 오류를 더 쉽게 확인할 수 있습니다.
